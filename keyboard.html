Web Accessibility Perspectives: Keyboard Compatibility

Introduction

Web accessibility is essential for people with disabilities and useful for all. Learn about the impact of accessibility and the benefits for everyone in a variety of situations.

Video on Keyboard Compatibility

[ Previous: none | Up: Overview | Next: Colors with Good Contrast ]

What is Keyboard Compatibility?

All functionality needs to work with the keyboard. That is, users can access and move between links, buttons, forms, and other controls using the Tab key and other keystrokes. Websites should not require a mouse; for example, pop-up calendars should also let users type in a date.

Who depends on this feature?

  • People with physical disabilities who cannot use the mouse.
  • People who are blind, and cannot see the mouse pointer on the screen.
  • People with chronic conditions, such as repetitive stress injuries (RSI), who should limit or avoid use of a mouse.

What are the additional benefits?

  • Keyboard interface works for people with temporary limitations, such as a broken arm or broken mouse.
  • Interface works in more situations, such as with on-screen keyboards and public terminals without a mouse.

What needs to happen for this to work?

Native HTML controls, like links, buttons, and form elements, work with the keyboard by default. Custom-made controls, CSS styles, and scripts that control interaction may need additional coding for keyboard compatibility. Markup the tab order so that navigation around the content and the controls is logical. Provide a way for users to jump between groups of related controls. Keyboard issues occur particularly in forms, menus, and applications with many controls.
